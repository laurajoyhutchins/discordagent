# Architecture Decision Records

This directory holds accepted architectural decisions and their rationale.

| Record | Status | Date |
|---|---|---|

Decision records explain why specific design choices were made. For the current documentation structure and how to add records, see the [contributing guide](../../contributing/repository-structure.md).

Durable design rationale that originated in planning documents (`docs/superpowers/`) has been migrated into the appropriate explanation pages. See the [architecture](../architecture/provider-neutral-runtime.md), [security](../security/trust-model.md), and [product](../product/why-discord-agent.md) pages for the canonical design explanations.
